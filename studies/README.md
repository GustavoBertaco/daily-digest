# Studies

A place to capture and share studies, notes, and brainstorms on topics I care about
— writing things down to think them through and to share knowledge with others.

## Topics

- [**Product Management**](./product-management/) — discovery, strategy, prioritization, metrics, ways of working.
- [**Data Platform**](./data-platform/) — architecture, ingestion, modeling, governance, tooling.
- [**Engineering**](./engineering/) — systems design, code, infrastructure, practices.

## Conventions

- One topic per subfolder. Add a new note as its own Markdown file.
- Name files in kebab-case, e.g. `metrics-that-matter.md`.
- Start each note with a title, a one-line summary, and the date.
- Use [`_template.md`](./_template.md) as a starting point for new notes.

## Adding a new study

1. Pick the relevant topic folder (or create a new one).
2. Copy `_template.md` to `<topic>/<your-title>.md`.
3. Fill it in and link it from this README if it's worth surfacing.
