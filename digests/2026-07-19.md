# 🤖 AI & Machine Learning

## AI Engineer

**[Your Agents Need a Save Button - Hamza Tahir, ZenML](https://www.youtube.com/watch?v=bZISsg7H7DA)**
Agents spend most of their runtime waiting — on a tool call, a human approval, the next step — while holding a live process awake and billing for compute. Hamza Tahir of ZenML argues that serializing agent state to durable storage (the "save button") eliminates idle compute cost and enables crash recovery without re-burning prior tokens.

- **The idle cost problem is structural.** Agents wait more than they work; keeping the process alive during waits means paying for compute that produces nothing, and that cost multiplies across every agent an organization wants to run overnight.
- **Freeze-to-zero is the mechanism.** The proposed pattern: serialize agent state to durable storage, drop compute to zero, and resume in milliseconds when there is work to do.
- **Crash recovery changes the retry economics.** Without durable state, a crash means re-executing every prior step from the top and re-paying for every token. A save point lets recovery resume from the last checkpoint.
- **Scale is the forcing function.** One idle agent is a rounding error; the math stops working when multiplied across an organization's parallel and overnight workloads.

**[Agents Need Feature Flags - Sachin Gupta](https://www.youtube.com/watch?v=zU4EagB311U)**
Most AI teams deploy behavior changes — prompts, tool access, model selection, memory policy, autonomy level — to 100% of users simultaneously, with no canary, no segment rollout, and no kill switch. Sachin Gupta argues agent infrastructure needs the same feature-flag discipline that web engineering standardized over a decade ago.

- **The blast radius is global by default.** Every agent deploy changes prompts, tools, model choice, memory policy, and autonomy level for all users at once — a configuration surface far wider than a typical web release.
- **Failures surface outside observability.** The snippet describes a "small" prompt tweak that broke 12% of users, with the failure discovered from a Discord screenshot rather than a monitoring alert.
- **Web engineering solved this in 2012.** Canary releases, per-segment rollouts, and kill switches are standard in web and mobile development; agent systems have almost none of it.
- **The flaggable surface for agents is wider.** Beyond code, the controllable dimensions include prompts, tool access, model selection, memory policy, and autonomy level — each a potential failure surface on any deploy.
