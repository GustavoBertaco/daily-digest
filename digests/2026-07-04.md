## AWS What's New

**[Amazon SageMaker Unified Studio now supports Terraform for provisioning](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-sagemaker-unified-studio-terraform/)**
SageMaker Unified Studio domains can now be provisioned via the open-source `terraform-aws-sagemaker-unified-studio` module, letting platform teams bring the service into existing version-controlled IaC pipelines. The launch enables consistent deployments across development, staging, and production accounts without leaving infrastructure tooling.

**[Amazon EC2 X8i instances are now available in additional regions](https://aws.amazon.com/about-aws/whats-new/2026/02/amazon-ec2-x8i-instances-ICN-KUL-NRT-region/)**
X8i instances — powered by custom Intel Xeon 6 processors exclusive to AWS and SAP-certified — expand to Asia Pacific (Seoul), Asia Pacific (Malaysia), and Asia Pacific (Tokyo). They deliver up to 43% higher performance, 1.5x more memory capacity (up to 6 TB), and 3.3x more memory bandwidth compared with comparable Intel cloud instances.

**[Amazon EC2 Dedicated Hosts now support AMD SEV-SNP](https://aws.amazon.com/about-aws/whats-new/2026/07/ec2-amd-sev-snp-dedicated-hosts)**
Confidential computing workloads can now run on single-tenant Dedicated Hosts with AMD Secure Encrypted Virtualization-Secure Nested Paging (SEV-SNP) enabled. Customers gain Dedicated Host benefits — controlled instance placement and host affinity — alongside hardware-level memory encryption on fully dedicated physical servers.

**[Amazon SageMaker HyperPod now supports AMI versioning and auto-patching](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-sagemaker-hyperpod-ami-version-auto-patch)**
HyperPod cluster administrators can now see which AMI versions are running across their clusters and receive automatic security patches without disrupting workloads. Previously, the lack of AMI visibility made drift hard to detect and security patching a manual, reactive process.

**[AWS Config now supports 8 new resource types](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-config-new-resource-types)**
AWS Config adds coverage for 8 new resource types across Amazon API Gateway, Amazon EC2, and Amazon S3 Vectors. Accounts with all-resource-type recording enabled will automatically begin tracking these additions; the types are also available as Config Rules targets for audit and remediation.

Full live stream of the closing day of AI Engineer World's Fair 2026 at Moscone West, San Francisco, covering keynotes and main-stage programming including sessions featuring Garry Tan, Mike Krieger, and talks on Harness Engineering and DSPy.

**[The Prompt Is Still a Punch Card - Ted Johnson, JoinIn AI](https://www.youtube.com/watch?v=hVJOnuhFmTA)**
Every major computing interface — punch cards, command lines, mice, menus, voice assistants, and now prompts — has been a brilliant compromise with the machines of its time, requiring humans to learn the machine's protocol. Ted Johnson argues AI is the first technology that gives us a genuine chance to renegotiate that bargain, and reframes it as an interface technology rather than merely an intelligence technology.

- **The historical pattern is the argument.** Computing history shows a consistent loop: a new machine arrives, humans learn its protocol, the protocol normalizes, and then a new interface renegotiates. Prompts are the current iteration of that loop, not the end of it.
- **Reframe AI as interface, not intelligence.** The talk explicitly shifts the evaluation lens from "what can the model know or do?" to "how does this change the human-machine negotiation of meaning?" That reframe changes what you optimize for when building AI products.
- **Prompts encode the machine's constraints, not the user's intent.** Prompt engineering is, by this argument, the modern equivalent of learning card-punch syntax — an adaptation humans make to fit the machine rather than the reverse. Recognizing this is the precondition for designing past it.
