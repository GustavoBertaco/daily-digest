## AI Engineer

**[RLM: Recursive Language Models for Large Codebases - Shashi, Superagentic AI](https://www.youtube.com/watch?v=8oyalrfwgjw)**
Large codebases break standard coding agents because context windows fill with tool output before architectural coherence can be maintained. RLM, drawn from an MIT paper, addresses this by loading the repo into a programmable REPL where the model writes code to inspect the codebase and recursively delegates focused sub-questions via `llm_query`.

- **Context overflow is the core failure mode.** As codebases scale, agents lose the architectural picture — not from a model capability ceiling but from how context is consumed by tool output.
- **RLM turns inspection into code.** Instead of streaming files into a context window, the model writes code to query the repo from within a REPL, keeping the working context lean and targeted.
- **Recursive delegation over monolithic reasoning.** Focused sub-questions are handed off via `llm_query`, enabling divide-and-conquer without exhausting the context budget on a single pass.
- **Inspectable trajectory by design.** The live demo covers end-to-end runs on both local and cloud models with fully visible trajectories — a practical requirement for debugging production agent behavior.

- **Minimal skill required dramatically widens the attacker pool.** Sophisticated vulnerability discovery no longer demands deep expertise, fundamentally changing who can mount serious attacks.

**[Semantic Blindness: 500,000 Sensors Confused an LLM - Raahul Singh & Vanč Levstik, Phaidra](https://www.youtube.com/watch?v=EUsPvBeIx70)**
Phaidra deployed state-of-the-art LLMs to manage real-time operations on industrial and AI factory infrastructure with 50,000 live sensors, deterministic dependencies, and thermodynamic constraints — and concluded that next-token prediction cannot solve combinatorial engineering problems.

- **Fundamental mismatch, not a tuning problem.** The core finding is structural: next-token prediction is the wrong tool class for deterministic combinatorial control, regardless of model quality or scale.
- **Real-world scale reveals what benchmarks hide.** Fifty thousand live sensors with thermodynamic and causal constraints expose failure modes that synthetic evaluations and toy deployments never surface.
- **Hard lesson from production, not simulation.** The team deployed to real industrial and AI factory environments and encountered the limits directly — this is field experience.
- **Deterministic systems require deterministic reasoning.** Physical infrastructure with causally closed constraints cannot be reliably managed by probabilistic generation.

**[The Agentic Web and the Bazaar Era of AI - Ramesh Raskar, MIT Media Lab](https://www.youtube.com/watch?v=sum9DgexFRQ)**
MIT Media Lab's Ramesh Raskar argues the current AI agent ecosystem — focused on memory, orchestration, and enterprise tooling — mirrors the early AOL era: closed platforms, proprietary agent stores, siloed orchestration layers. The next transformation will require open infrastructure for agents to discover, transact, and co-learn across organizational boundaries.

- **We are in the AOL era of AI agents.** Closed, proprietary, siloed platforms are a phase to move through, not a destination — the historical analogy frames the urgency of open infrastructure.
- **Current priorities are scaffolding, not the endpoint.** Memory, orchestration, and enterprise deployment are first-order concerns on the path to a genuinely open agentic ecosystem, not the ecosystem itself.
- **The Agentic Web requires cross-boundary operation.** Discovery, transacting, and co-learning across organizations are capabilities that today's proprietary agent stores explicitly prevent.

**[ReviewDebt: a practical framework for scoring every pull request — Sachin Gupta, Ebay](https://www.youtube.com/watch?v=TJPInBjhE4Q)**
Sachin Gupta from eBay argues that the metrics teams use to evaluate AI coding agents — PR count, lines of code, cycle time, developer NPS — miss the hidden costs accumulating as ReviewDebt: bloated diffs, weak tests, ambiguous rationale, ownership sprawl, and reviewer time that now exceeds what was previously spent writing code.

- **Speed-trust gap is the core problem.** Agents ship PRs faster than humans can confidently review them, generating debt that doesn't appear in any standard engineering dashboard.
- **Current metrics are structurally blind to the real cost.** PR count, lines of code, cycle time, and NPS measure throughput but miss diff bloat, test quality, ownership sprawl, and reviewer burden.
- **ReviewDebt names and scores what goes unmeasured.** The framework makes hidden costs visible before they consume engineering velocity — measuring what existing tooling ignores.
- **Reviewer burden has inverted.** Human reviewers now spend more time verifying AI-written code than they used to spend writing their own — the efficiency gain has been transferred upstream.
