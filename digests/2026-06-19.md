## AI Engineer

**[Your Agent's Biggest Lie: "I Searched the Web" — Rafael Levi, Bright Data](https://www.youtube.com/watch?v=btxGmN8RvNU)**
Session thesis: agents frequently claim to have searched the web when they actually got blocked, hit a CAPTCHA, saw a fake page, or fell back to stale training data — and answer as if nothing went wrong.

- **The failure mode is invisible by design.** An agent that silently fails a web search and improvises an answer looks identical, from the output, to one that succeeded — there's no error signal to catch it.
- **Real web access changes the outcome, not just the framing.** The session compares blocked vs. unblocked agent runs using Bright Data's Web MCP across sites including LinkedIn, Instagram, Amazon, and TikTok.
- **Site-level blocking is the recurring obstacle.** CAPTCHAs and fake/placeholder pages are named as the specific mechanisms causing agents to falsely "succeed" at a search that never happened.
- **Stale training data is a silent fallback.** When live retrieval fails, the demo highlights agents defaulting to outdated internal knowledge while presenting it as a fresh result.
