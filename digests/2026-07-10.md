## AWS What's New

**[AWS Builder Center Now Offers Free Sandbox Environments](https://aws.amazon.com/about-aws/whats-new/2026/07/aws-builder-center-sandbox/)**
AWS Builder Center now provides free, time-limited sandbox environments for eligible workshops, removing the need for a personal AWS account, credit card, or concern about unexpected charges. Builders of all skill levels can deploy resources, write code, and experiment in a pre-provisioned AWS account — opening workshops to participants who previously couldn't engage due to billing requirements.

**[AWS Security Hub now offers Network Scanning to identify publicly reachable resources](https://aws.amazon.com/about-aws/whats-new/2026/07/aws-security-hub-network-scanning/)**
AWS Security Hub adds active Network Scanning that probes resources from the public internet to detect actual reachability — not just theoretical exposure inferred from security group rules and route tables. It discovers public IPs, virtual machines, and load balancers across both AWS and Azure environments, and identifies reachable ports and running services.

**[Amazon Redshift RG instances now available on the trailing track](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-redshift-graviton-rg-instances-trailing-track)**
As of July 7, 2026, rg.4xlarge and rg.xlarge Graviton-based instance types are available on the Redshift trailing track (P201), which targets production workloads requiring the stability of a version already validated on the leading track. RG instances are now available on both tracks.

**[Amazon Aurora DSQL change data capture (CDC) Is now generally available](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-aurora-dsql-cdc-ga/)**
Aurora DSQL CDC is now generally available, enabling real-time streaming of insert, update, and delete events to Amazon Kinesis Data Streams with no infrastructure to manage. Supported use cases include cross-service data synchronization, Lambda triggers, and downstream data integration pipelines.

**[Amazon Connect Customer now supports forecasting, planning, and scheduling for Tasks and Emails](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-connect-customer-agent-scheduling-tasks/)**
Amazon Connect now extends workforce forecasting, capacity planning, and scheduling to Tasks and Emails channels alongside existing Voice and Chat support. The system accounts for channel-specific characteristics — concurrent work handling, task durations ranging from minutes to months, and distinct service-level requirements — so workforce plans reflect how operations actually run.

## AI Engineer

**[What do we build now? — Theo Browne, @t3dotgg](https://www.youtube.com/watch?v=xUnRQ9vLXxo)**
Theo Browne's closing keynote at AIEWF 2026 argues that AI's architectural arc — from tool-calling through long-running task execution to full orchestration — demands a corresponding shift in how engineers design products. The risk is building AI-era software in the shape of the previous era's patterns.

- **Reject skeuomorphism.** Browne's central warning: most teams are wrapping AI in the visual and interaction design of old software rather than building for what models can actually do now.
- **Three generations of capability, three different mental models.** The snippet traces a clear arc — tool-calling (Sonnet 3.5) → long-running task execution (Opus 4.5) → orchestration (Mythos) — each requiring engineers to rethink product architecture from scratch.
- **Think wider, not just bigger.** The keynote's provocation: the constraint is not model capability, it is the imagination of the engineers building on top of it.
- **The default approach is already outdated.** AI model evolution now outpaces typical product iteration cycles, meaning "add AI to the existing product" has a short shelf life as a strategy.

**[Think You Can Build a Game with AI? Think Again! - Danielle An & David Hoe, Meta](https://www.youtube.com/watch?v=grdoOC1BT1s)**
Meta's Danielle An and David Hoe argue that AI-assisted game creation — vibe-coding a game with Gemini on Three.js — is already table stakes, and the genuinely interesting design space is what comes next once that capability is fully commoditized.

- **Vibe coding is no longer a differentiator.** The talk opens by treating AI-assisted game prototyping as something anyone can already do — the competitive question is what skill looks like after that floor rises.
- **The next massive hit is just about to be born.** The talk frames this as an imminent opportunity — the game that millions of people across the world will play is almost here, for whoever figures out what comes next.
- **The window is right now.** The "just about to be born" language signals that first-mover timing in this space is the relevant variable to optimize for.

**[Your agent is blindfolded — Johan Lajili, Poolside AI](https://www.youtube.com/watch?v=iRcX54EO5g8)**
