## AWS What's New

**[Amazon EC2 C7a instances are now available in the US West (N. California) Region](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-ec2-c7a-instances-us-west-ncalifornia-region/)**
C7a instances — powered by 4th Gen AMD EPYC (Genoa) processors at up to 3.7 GHz with AVX-512, VNNI, and bfloat16 support — are now available in the US West (N. California) region, delivering up to 50% higher performance compared to C6a instances. They introduce DDR5 memory with 2.25x more memory bandwidth than the prior generation.

**[Amazon EC2 M8a instances now available in the Asia Pacific (Hyderabad) region](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-ec2-m8a-instances-asia-pacific-hyderabad-region/)**
General-purpose M8a instances, powered by 5th Gen AMD EPYC (Turin) processors at up to 4.5 GHz, are now available in Asia Pacific (Hyderabad), delivering up to 30% higher performance, up to 19% better price-performance, and 45% more memory bandwidth compared to M7a instances. The increased memory bandwidth targets latency-sensitive workloads.

**[AWS Network Load Balancer now supports Listener Rules for custom traffic routing](https://aws.amazon.com/about-aws/whats-new/2026/07/aws-network-load-balancer-supports-listener-rules/)**
NLB now supports listener rules that route connections to different target groups based on source IP address type, letting a single dual-stack NLB direct IPv6 client traffic to IPv6 targets and IPv4 client traffic to IPv4 targets while preserving the original client IP end to end. This eliminates the previous requirement to run two separate load balancers — one per address family — to serve both IPv4 and IPv6 clients.

**[AWS Entity Resolution now supports advanced real-time matching](https://aws.amazon.com/about-aws/whats-new/2026/07/aws-entity-resolution/)**
AWS Entity Resolution's GenerateMatchId API now supports real-time matching with advanced rulesets — including Exact, ExactManyToMany, and AND/OR operators — enabling record matching in milliseconds. Previously, advanced rulesets were batch-only and could take minutes to hours; only simple rule-based workflows were available for real-time use.

**[AWS Lambda durable functions now supports customer managed key encryption](https://aws.amazon.com/about-aws/whats-new/2026/07/durablefunctions-cmk/)**
Lambda durable functions — which provide automatic state management for long-running workflows written directly in Lambda code — now supports encryption of execution state with AWS KMS customer managed keys. The default AWS-owned key encryption remains in place; the CMK option is aimed at regulated industries that require direct control over their encryption keys.

**[Amazon Corretto July 2026 Quarterly Updates](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-corretto-july-2026-quarterly-updates)**
Amazon Corretto July 2026 quarterly updates cover security and critical fixes across LTS and Feature Release versions: Corretto 26.0.2, 25.0.4, 21.0.12, 17.0.20, 11.0.32, and 8u502 are now available for download. Starting with this release, the default Corretto Docker images are based on Amazon Linux 2023, while Amazon Linux 2 images continue to be provided separately.

**[AWS Organizations increases RCP quota to 2,000 per organization](https://aws.amazon.com/about-aws/whats-new/2026/07/aws-organizations-resource-control-policy-limit-increase-2000)**
AWS Organizations now supports up to 2,000 resource control policies (RCPs) per organization, doubling the previous limit of 1,000. RCPs enable centralized management of the maximum permissions available to resources across member accounts; the higher quota is targeted at large multi-account environments that need more granular resource access definitions.

**[Amazon EKS now supports EFA and placement groups on Amazon EKS Auto Mode and Karpenter](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-eks-efa-placement-groups/)**
EKS Auto Mode and the open-source Karpenter project now support EC2 placement groups and Elastic Fabric Adapter (EFA) network device configuration for node pools, enabling control over both EFA interface configuration and how EC2 instances are physically distributed across AWS infrastructure. These capabilities target distributed training and inference workloads that require precise network and placement configuration.

# 🤖 AI & Machine Learning

## AI Engineer

**[AI on Your Lakehouse: Context Comes in Shapes, Not Queries — Zach Blumenfeld, Neo4j](https://www.youtube.com/watch?v=kRkcNOsRyYg)**
The failure mode Blumenfeld targets is not retrieval failure but relevance failure: an agent that can reach your data via vector search or Text2SQL can still return confident, wrong answers because neither method tells it what is relevant or how the pieces connect.

- **Retrieval is solved; relevance isn't.** Vector search hands the agent a slice and Text2SQL hands it another, but neither encodes the structural relationships that determine what actually matters for a given question.
- **Context comes in shapes.** The missing layer is graph-shaped context — not a better query, not a better model.
- **Three reusable graph shapes on top of lakehouse data.** Blumenfeld builds these with Neo4j in a hands-on workshop, each shape designed to answer a class of question that flat retrieval cannot handle correctly.
- **The graph layer complements, not replaces, existing access patterns.** Vector and SQL retrieval remain; the graph provides structural context that disambiguates what retrieved data means.

**[Why We Killed Our Multi-Agent Pipeline — Subbiah Sethuraman and Abhilash Asokan, ZS Associates](https://www.youtube.com/watch?v=u6jJcIFDLE4)**
ZS Associates built a pharma analytics pipeline that decomposed the analyst workflow across four specialized agents wired to an orchestrator — and discovered that architecturally clean decomposition produces wrong outputs when no single agent owns the full reasoning chain from cause to appropriate action.

- **Right cause, wrong action.** The pipeline correctly identified that prescriptions dropped 18% in a territory because a payer moved the drug to a worse formulary tier — then recommended sending more sales reps, the wrong intervention, because no agent connected the diagnosis to the right response.
- **Specialization severs the reasoning chain.** Splitting detection, localization, cause-finding, and synthesis across separate agents means synthesis only receives outputs, not the reasoning that produced them.
- **They killed the pipeline.** The team scrapped the multi-agent architecture rather than patching the orchestration layer; the replacement centers on a model where full context is preserved end to end.
- **The lesson is architectural, not incidental.** Task boundaries that appear clean in a diagram can eliminate exactly the reasoning continuity the task requires — defaulting to decomposition carries a real cost.

**[Citation Needed: Provenance for LLM-Built Knowledge Graphs — Daniel Chalef, Zep AI](https://www.youtube.com/watch?v=H7puB0RwJMM)**
When an LLM synthesizes facts from multiple sources into a knowledge graph, source attribution dissolves — and downstream consumers have no way to verify where a fact came from, whether it is still current, or which source to trust when sources conflict.

- **Source IDs don't survive synthesis.** Stamping source identifiers before LLM processing doesn't preserve them: the model merges entities from an EHR record, a lab report, and patient-entered intake data, collapsing distinct attributions into a single synthesized fact.
- **Graphs are dynamic; provenance pointers go stale.** Later data can invalidate earlier facts, so a provenance pointer to a "source" in an evolving graph can reference something that no longer reflects current reality.
- **Provenance must be a first-class data model citizen.** Chalef's argument is that provenance cannot be retrofitted after the graph is built — it must be designed in from the start as a structural property, not an annotation.
- **The medical case makes the stakes concrete.** A clean, confident penicillin allergy fact with no traceable source is not a data quality concern — it is a patient safety concern.

**[Local Agentic Theory For Mobile Games — Shafik Quoraishee & Joanne Song, The New York Times](https://www.youtube.com/watch?v=418t26CVz-w)**
Running agents entirely on-device with no cloud calls imposes a hard real-time budget that forces agent design to be genuinely efficient rather than deferring compute to a server.

- **Sixteen milliseconds is the ceiling.** The Space Invaders agent must complete its full perceive-predict-act loop within one 16ms render frame; every design decision becomes a latency decision.
- **Constraint satisfaction for language tasks.** The crossword agent uses a constraint graph with backtracking — a classical AI approach that fits the tight compute budget better than generative inference at action time.
- **The Times is explicit about human authorship.** These are experimental systems; the organization states clearly that its actual puzzles are made by people with no AI involvement in the games.
- **On-device forces resource discipline by design.** Without a server to offload to, the architecture must be structurally efficient — a constraint that surfaces trade-offs that cloud-based agent work can defer indefinitely.

**[Video Has No Memory. Here's How We Built One. — James Le, TwelveLabs](https://www.youtube.com/watch?v=mOf-PP4mVjA)**
Almost all video AI systems answer each query from scratch with no persistent understanding of content they have already processed — Le argues the core constraint is not context window size but the absence of a memory layer over video.

- **Stateless by default.** Nearly every video AI system re-processes footage from scratch on each query; there is no accumulated, indexed understanding to query against.
- **A bigger context window doesn't fix it.** The real problem is not that the model can't hold enough video — it is that there is no persistent representation of what the video contains that can be queried relationally.
- **The capability gap is made concrete by the use case.** Querying 67 videos from the 2022 World Cup for near misses with per-shot reasoning, or tracking Messi's movement and camera framing across an entire corpus, requires cross-video relational memory that stateless models structurally cannot provide.
- **TwelveLabs's approach centers on persistent video indexing.** The solution treats video as a corpus to be indexed into a queryable memory layer, not a stream to be re-ingested per question.

**[Why Agentic Systems Need Ontologies — Frank Coyle, UC Berkeley](https://www.youtube.com/watch?v=Sir59K8ZDPU)**
Coyle argues that agent failures — double refunds, misrouted payouts, ambiguous order statuses — are not prompt engineering problems but structural symptoms of operating without a formal ontology as a logical constraint layer outside the model.

- **Probabilistic reasoning needs deterministic guardrails.** LLMs reason probabilistically over domains they only partially understand; a formal ontology sitting outside the model enforces logical constraints that instructions inside a prompt cannot reliably guarantee.
- **The failures are structural, not incidental.** A second refund on the same order, a payout sent to the support desk instead of the buyer, an order status of "probably shipped" — these are outputs from a model with no formal encoding of business rules.
- **The ontology lives outside the model.** Coyle places it as an external layer rather than part of the prompt, so it acts as a hard logical guardrail the model cannot probabilistically override.
- **Brittle tools and fragile handoffs share a root cause.** When tool interfaces and handoff contracts are under-specified, agents fill gaps probabilistically; a formal ontology narrows the space of valid completions to what is semantically legal.

**[Learned Execution Graphs for Anomaly Detection & Drift in APIs — Ritvik Pandya, JP Morgan Chase](https://www.youtube.com/watch?v=u1yaOeEX4e8)**
JP Morgan's API monitoring reported the system healthy — latency down, errors at zero — while a mandatory processing step was being silently skipped; modeling each request as a learned execution graph exposed structural deviations that health-metric aggregates are blind to.

- **Conventional metrics lie by omission.** Latency was down and errors were at zero, yet a required middleware step was quietly absent — nothing in the standard monitoring stack caught it.
- **Each request becomes a DAG.** The team models the middleware steps traversed by each API call as a directed acyclic graph, learning the expected shape from telemetry at over 1,600 requests per second.
- **Anomaly is structural, not statistical.** A skipped, reordered, or injected step is detected by comparing the observed DAG against the learned baseline; no aggregate metric threshold needs to cross for the deviation to become visible.
- **The graph catches what aggregates hide.** Because the model operates on execution topology rather than aggregated statistics, a single missing middleware step is detectable even when system-level throughput and latency look normal.

**[From Systems of Record to Systems of Context — Omri Bruchim & Tomer Ast, monday.com](https://www.youtube.com/watch?v=Btk8wDUVs74)**
AI assistants fail at actionable prioritization not because they lack data, but because a system of record stores what happened without encoding what it means — monday.com is rebuilding its product as a system of context.

- **The retrieval problem is solved; the meaning problem is not.** Full access to every board, task, email, and Slack message is not the bottleneck; the gap is that a system of record provides no structure for what any of it signifies.
- **A concrete failure makes the gap visible.** When asked what to focus on right now, the AI assistant told Bruchim to go to the gym — a response that is only possible when a system has data but no context about what it means for work.
- **Context is what gets built, not retrieved.** monday.com's approach is to construct a representation layer that captures significance, relationships, and intent — not just event history.
- **This is a data model shift, not a model swap.** The change is in what gets stored and how it is structured; the underlying LLM is not the variable being changed.

**[Your Moat Is Your Data Model — Mike Phipps, Gates Foundation](https://www.youtube.com/watch?v=jt1Pbr_n6oU)**
As models, frontends, and agent frameworks commoditize, Phipps argues the durable advantage is the data model — the structured encoding of institutional knowledge about how questions in a domain are supposed to be answered.

- **Commoditization is the baseline assumption.** Everything except the data model will be available to competitors on equivalent terms; the durable moat is the structured knowledge unique to an organization.
- **25 years of grantmaking encoded in one graph.** The Gates Foundation modeled its entire history — over $7 billion a year across 2,000 grants and 4,000 people — into a single Neo4j knowledge graph served to Claude through one MCP server.
- **Built for agents, not dashboards.** The graph is structured for agent traversal rather than human-readable reporting — the design assumption is that the consumer is an agent, not a dashboard reader.
- **One MCP server as the entire interface.** Rather than exposing a suite of APIs or data endpoints, the complete knowledge graph surfaces through a single MCP server, simplifying agent integration while preserving the model's structure.

**[Active Graph Agent Runtime (BabyAGI 4) — Yohei Nakajima, Untapped Capital](https://www.youtube.com/watch?v=khVX_BUnEwU)**
Nakajima, the creator of BabyAGI, inverts the standard agent architecture: instead of centering the LLM and bolting on memory and logging, ActiveGraph centers an immutable event log and derives the agent's state from it — making the log the agent rather than a side effect of running one.

- **The log is the agent.** Every action and state change flattens into a single typed immutable event log; the agent's current state at any point is a projection of that log, not a separate mutable object.
- **Fault recovery becomes a log replay.** When an API key failed at question 350 of a 500-question eval, the agent rolled back one step and resumed at question 353 — because resumption is replaying the log from the last valid state.
- **The architecture inverts the mainstream.** Standard agent design puts the LLM at the center and adds memory and logging as features; ActiveGraph puts the event log at the center and derives everything else from it.
- **Reliability is structural, not defensive.** The fault-recovery properties are built into the foundational architecture rather than added as error handling or retry logic after the fact.
