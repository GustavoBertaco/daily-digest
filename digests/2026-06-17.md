## AWS What's New

**[AWS Transform now supports Amazon FSx for NetApp ONTAP (Public Preview)](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-transform-vmware-fsx-for-ontap-preview)**
AWS Transform's agentic migration service now adds Amazon FSx for NetApp ONTAP as a target for block storage migrations, alongside the existing Amazon EBS option. The service automates discovery, planning, and migration of workloads from on-premises or cloud sources.

**[AWS Management Console Private Access now works without internet connectivity](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-management-console-private/)**
Console Private Access now lets customers reach the AWS Console from VPCs with no internet connectivity at all, useful for air-gapped environments. Previously it could restrict access to authorized accounts/networks but still required an internet path.

**[Grok 4.3 from xAI now available in Amazon Bedrock](https://aws.amazon.com/about-aws/whats-new/2026/06/grok-amazon-bedrock/)**
xAI's Grok 4.3 joins Amazon Bedrock as a model provider, adding a reasoning-first model with always-on, configurable reasoning effort (none/low/medium/high). AWS says the always-active reasoning makes it behave more consistently across multi-step agent loops.

**[Amazon Bedrock AgentCore Memory now supports strictly consistent metadata for long-term memory](https://aws.amazon.com/about-aws/whats-new/2026/05/agentcore-memory-scmetadata)**
AgentCore Memory now lets applications attach metadata values directly rather than relying solely on LLM inference during extraction, so metadata passes through extraction and consolidation exactly as supplied. This improves reliability of organizing, filtering, and routing long-term memory records.

**[Amazon FSx for OpenZFS now supports on-demand data replication across AWS opt-in Regions](https://aws.amazon.com/about-aws/whats-new/2026/06/on-demand-cross-region-replication/)**
FSx for OpenZFS now supports on-demand replication of incremental point-in-time snapshots into AWS opt-in Regions (not enabled by default). This targets disaster recovery, cross-region/account replication, and lower-latency access for global users.

**[Amazon CloudWatch introduces Log Analytics for unified log analysis](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-cloudwatch-log-analytics/)**
CloudWatch Log Analytics unifies Logs Insights querying, Live Tail real-time streaming, and Contributor Insights into one console experience. Customers can run multiple queries across tabs while keeping existing features like patterns, saved parameterized queries, and facets.

**[Amazon Route 53 Resolver DNS Firewall now supports Palo Alto Networks Advanced DNS Security (Preview)](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-route-53-resolver-dns/)**
Route 53 Resolver DNS Firewall can now enforce Palo Alto Networks Advanced DNS Security threat protections directly via DNS Firewall rules. Customers subscribe to PANW through an embedded AWS Marketplace widget, avoiding separate firewalls or VPC reconfiguration.

**[Amazon ECS Express Mode is now available in AWS GovCloud (US) Regions](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-ecs-express-mode-govcloud/)**
ECS Express Mode is now available in AWS GovCloud (US-East) and (US-West). It lets developers rapidly launch containerized web apps/APIs, with every Express Mode service automatically receiving an AWS-provided domain name.

**[AWS launches Cost Explorer historical data retention for accounts in billing groups](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-cost-explorer/)**
Cost Explorer now retains historical billing data for accounts mapped into billing groups via AWS Billing Conductor/Billing Transfer. Previously, mapping an account into a billing group restricted access to its historical billing-rate data.

**[AWS Lambda Managed Instances now supports Tag Propagation for Managed Resources](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-lambda-managed-instances-tag-propagation/)**
Lambda Managed Instances now propagates tags automatically to managed resources like EC2 instances, EBS volumes, and ENIs. This helps enforce cost allocation, service control policies, and compliance across resources provisioned by capacity providers.

## AI Engineer

**[You Might Not Need 50 Diffusion Steps — Ziv Ilan, Nvidia](https://www.youtube.com/watch?v=gHs5ZiY80PM)**
Ziv Ilan's thesis is that the standard 50-step diffusion denoising process is mostly unnecessary overhead, and that combining quantization, caching, and step distillation can get video diffusion close to real time on a single GPU without a new architecture.

- **The win came from removing steps, not from a new architecture.** Ilan's team got a video diffusion model running near real time on a single NVIDIA Blackwell B200 by stripping down the default 50-step denoising process rather than redesigning the model.
- **Step distillation does the heavy lifting.** A student model is trained to match a teacher's output using far fewer steps — four, eight, or in some cases just one — instead of the full fifty.
- **It's a layered optimization stack, not one trick.** Dynamic quantization and caching are combined with step distillation, with each layer contributing part of the overall speedup.
- **Single-GPU real-time inference is the practical target.** The result is framed around making video diffusion viable on one Blackwell B200, rather than requiring multi-GPU infrastructure.
