**[Stop Evaluating Models Like It's the 50s - Alejandro Vidal, Mindmakers](https://www.youtube.com/watch?v=EfcfUB2uprc)**
AI benchmarks still run on Classical Test Theory — count right answers, weight every question equally, report the percentage — while psychometrics developed far more powerful measurement frameworks for exactly this problem decades ago and those frameworks have gone unused.

- **Classical Test Theory is the wrong model for AI.** Treating every benchmark question as equally diagnostic ignores that some items discriminate between capability levels far better than others, which distorts the signal.
- **Item Response Theory (IRT) was built for this.** IRT — the math underlying the SAT and GRE — models each test item's difficulty and discriminating power, producing richer measurement from the same questions without requiring more of them.
- **Decades of prior art sit unused.** Psychologists spent a century learning to measure something invisible and uncooperative; the argument is that AI evaluation should borrow those conclusions rather than repeat the early mistakes.

**[What Does Done Even Mean? Agents and Paperclip's Liveness Model - Dotta, Paperclip](https://www.youtube.com/watch?v=7P0elyLIxXo)**
When agents produce work faster than humans can review it, the informal meaning of "done" breaks down — Paperclip's liveness model reframes completion as a formal trust protocol rather than a social convention between teammates.

- **"Done" must carry evidence.** An artifact is complete when it meets a stated standard, carries evidence of that, has been reviewed by the right verifier, assigns ownership of remaining risk, and explicitly authorizes the next action — not before.
- **Approval theater is the primary failure mode.** When volume exceeds reviewers' capacity, rubber-stamp sign-offs accumulate and "done" becomes meaningless; the liveness model is designed to prevent this without slowing throughput.
- **Verifier identity is part of the specification.** "Checked" means checked by the *right* verifier — the model requires specifying who or what has authority to close a task, so that authorization is traceable.
- **Completion must gate the next action.** A done artifact without a clear authorization for what follows resets every handoff to ambiguity and stalls the work-in-progress pipeline.

**[Every Solo Agent Builder Eventually Reinvents a Worse Version of CI/CD - Sumaiya Shrabony](https://www.youtube.com/watch?v=WLXxTaPagA8)**
Solo agent builders independently rediscover regression testing, scheduled runs, schema contracts, rollback, and output validation — and produce worse, narrower versions of each; the argument is to adopt existing software engineering infrastructure rather than treating agents as a novel category that needs new primitives.

- **Five reinventions, all already named.** Testing agent output after a change = regression testing. Scheduled runs with failure alerts = CI. Preventing schema changes from silently breaking downstream skills = contract testing. Recovering from bad outputs = rollback. Pre-production checks = staging.
- **Silent schema drift is the most dangerous failure.** A change to one skill's output schema silently breaks three downstream skills — the same dependency problem package managers and API contracts were designed to prevent.
- **Reinventing alone produces a worse result.** The cost is not just duplicated effort — it is narrower, more brittle tooling built without the decades of real-world iteration the originals carry.

After 1,000 hours orchestrating autonomous agent fleets under human direction at Snapchat, the consistent finding is that the bottleneck is not the model but the harness — agents that succeed on the first run routinely fail on subsequent iterations because the surrounding loop lacks persistent memory and contextual guardrails.

- **The model is not the gap.** The failure pattern across 1,000 hours of autonomous fleet orchestration consistently traces back to the harness — the loop, memory, and guardrails wrapped around the model — not the model itself.
- **First-run success is a false signal.** Agents that handle complex tasks on the first iteration routinely fail on re-runs because each invocation starts without the accumulated context the harness should be maintaining.
- **Persistent memory is a structural primitive.** Without cross-invocation memory, agents operate blind to prior outcomes, making reliable iteration structurally impossible regardless of model capability.
- **Contextual guardrails enable deterministic behavior.** The talk frames guardrails not as safety constraints but as the mechanism that converts non-deterministic models into predictable, reusable teammates.
