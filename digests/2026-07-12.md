## AI Engineer

**[remobi.app: Don't change your terminal workflow for mobile](https://www.youtube.com/watch?v=5192csoTkVo)**
remobi.app is a mobile interface for developers that keeps the terminal workflow intact rather than replacing it. Users swipe between running agents and unblock when one is stuck.

**[Stop Evaluating Models Like It's the 50s — Alejandro Vidal, Mindmakers](https://www.youtube.com/watch?v=EfcfUB2uprc)**
AI evaluation still defaults to Classical Test Theory — count correct answers, weight every question equally, report a percentage — while ignoring decades of psychometric research that built more rigorous measurement tools for exactly this kind of problem.

- **Classical Test Theory's core flaw.** Treating every benchmark question as equally informative collapses hard and easy items into a single percentage, making scores incomparable across different question sets.
- **Item Response Theory as the upgrade.** IRT (the math behind the SAT and the GRE) models each item individually rather than treating every question as equally informative — borrowing this framework from measurement science changes the picture.
- **A century of measurement science, unused.** Psychologists developed IRT to measure something invisible and uncooperative — human intelligence. AI evaluation is repeating early mistakes that field already resolved.
- **Rankings may be wrong.** Applying a more rigorous scoring model changes which models look strongest, meaning current leaderboards may be misranking models due to a fundamentally flawed measurement framework.

**[What Does Done Even Mean? Agents and Paperclip's Liveness Model — Dotta, Paperclip](https://www.youtube.com/watch?v=7P0elyLIxXo)**
When agents produce more output than humans can realistically review, "done" becomes a liability without a formal protocol — Paperclip's liveness model redefines task completion as an artifact that meets a stated standard, carries evidence, assigns ownership of remaining risk, and explicitly authorizes the next action.

- **Done is a trust protocol, not a state.** Completion means the artifact satisfies a checkable standard and has been verified by the appropriate party — not that an agent emitted a final message.
- **Approval theater as the failure mode.** Without a liveness model, human reviewers rubber-stamp agent output they cannot realistically evaluate, creating the illusion of oversight without its substance.
- **Explicit authorization prevents assumption.** Each completed artifact must clearly state what action it authorizes next, stopping agents from treating silence as permission to proceed autonomously.
- **Assigning residual risk is part of done.** Surfacing what the agent could not verify — rather than silently omitting it — is a requirement for a task to be legitimately complete and ownership to be clear.

Running OpenClaw without hardening access is a security liability; this talk covers contributing trusted-proxy authentication mode to the upstream project and building Clawspace, a browser-based file explorer and editor for OpenClaw workspaces.

**[Stop AI Agent Hallucinations: 5 Techniques + Production Patterns — Elizabeth Fuentes, AWS](https://www.youtube.com/watch?v=vJukHCIv7Ck)**
Agent hallucinations in production are not prompt engineering failures — they are architectural limitations requiring structural solutions, illustrated by concrete failure patterns: overbooking, fabricated statistics, and costly wrong-tool selection.

- **Failure modes are architectural, not tonal.** An agent booking 15 guests in a 10-person room, inventing statistics when data is absent, or picking the wrong tool from 29 options are not wording issues — they expose gaps in how agents access and validate information.
- **Graph-RAG addresses relationship-dependent queries.** Replacing vector similarity search with Neo4j Graph-RAG models entity relationships precisely, yielding 73% fewer fabricated results in the described benchmark.
- **Tool selection at scale is a disambiguation problem.** Choosing among 29 tools without structural guidance leads to wrong selections that waste $47 in tokens per failure — a measurable cost that compounds at production scale.
- **Five research-backed techniques, not heuristics.** The workshop frames each intervention as grounded in research rather than trial-and-error, positioning hallucination mitigation as principled engineering.

A third-generation Indian machinery company built Ira, a 39-agent operating system covering sales, business development, recruitment, quoting, marketing, and organizational memory — without relying on any off-the-shelf agent framework, orchestrated by a dedicated Athena agent through a 17-stage request pipeline.

**[Every Solo Agent Builder Eventually Reinvents a Worse Version of CI/CD — Sumaiya Shrabony](https://www.youtube.com/watch?v=WLXxTaPagA8)**
Solo agent builders working long enough in isolation will independently recreate five software engineering primitives — output verification, scheduling with failure detection, schema-change detection, rollback, and pre-production validation — without recognizing them as the CI/CD practices software engineering solved decades ago.

- **Five reinventions, each with a name.** Verifying agent output maps to regression testing; scheduled runs with failure alerts maps to cron plus monitoring; preventing downstream schema breaks maps to contract testing; recovering from garbage runs maps to versioned rollback; pre-production checks map to staging gates.
- **Schema changes are the silent failure.** One agent's output schema change breaking three downstream agents without warning is precisely the breaking-change problem that contract testing was designed to catch before it propagates.
- **Not naming the pattern is costly.** Builders who don't recognize they are reinventing CI/CD won't look for existing tooling, ending up with bespoke, fragile implementations of already-solved problems.
- **The fix is conceptual, not technical.** Recognizing that agent orchestration has the same pipeline-reliability requirements as software delivery unlocks an existing body of tooling and accumulated practice.

**[Develop at Idea Velocity — Jeffrey Lee-Chan, Snapchat](https://www.youtube.com/watch?v=9arM9b7JgOo)**
After 1,000 hours orchestrating autonomous agent fleets under human direction, the consistent finding is that the bottleneck is not model capability but the surrounding harness: agents that complete complex tasks on the first run routinely fail on subsequent iterations because the loop lacks persistent memory and contextual guardrails.

- **The harness is the bottleneck.** The biggest leverage in production multi-agent systems is the scaffolding — how context is preserved, how guardrails are enforced, how the loop around the model is structured — not the underlying model itself.
- **Persistent memory prevents iteration failure.** Without memory of prior decisions and state, each subsequent run starts from degraded context and loses the coherence that made earlier runs succeed.
- **Contextual guardrails enable determinism.** Preventing agents from drifting outside expected behavior as tasks evolve requires explicit constraints built into the harness, not implicit trust in model consistency.
- **Idea velocity as the design target.** The goal is collapsing the gap between conceiving a task and executing it reliably — moving from idea to trusted agent action, not just from prompt to API call.
