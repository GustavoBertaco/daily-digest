## AI Engineer

**[The Pipeline Is Dead - Iris ten Teije, Sky Valley Ambient Computing](https://www.youtube.com/watch?v=bRnoEpoK5m4)**
The classic software distribution model — one canonical version per release, the same for every user — was shaped by the cost of producing a change. Iris ten Teije argues that cost is now effectively zero, changes can happen at runtime in the user's session, and the boundary between distribution and development is dissolving; the infrastructure stack has not yet caught up.

- **One-version-per-release was always an economic constraint, not a technical law.** The assumption that everyone runs the same build existed because producing a change was expensive. That constraint is gone, and the model built on it is collapsing.
- **Truth becomes local when every user runs a different version.** Systems that assume a single canonical state break when versions are per-session. Where truth lives, and how it is authoritative, must be re-thought at the session or user level.
- **Debugging semantics change entirely at runtime.** How you debug a program that exists for one person in one session is a different problem than debugging a reproducible global build — traditional observability tools built for shared versions do not fit.
- **Infrastructure is the lagging constraint.** The capability to generate per-user runtime variants already exists; the infrastructure to operate, audit, and support those variants reliably does not yet.

**[500 people vibe-coded for 30 days. I was one of them. - Sanja Grbic, Automattic](https://www.youtube.com/watch?v=UcYoMg-8-L8)**
Automattic — the company behind WordPress — ran a company-wide 30-day experiment called Radical Speed Month: pause the roadmap and see how fast real software could ship. Sanja Grbic, a product designer with over a decade of experience, shipped three products that month and uses the experience to describe a new collaboration model between designers and engineers when traditional bottlenecks disappear.

- **Pausing the roadmap is the experiment.** By suspending planned work and giving 500 people latitude to ship, Automattic created the conditions to measure true organizational velocity — a controlled bet on what the team could do unconstrained.
- **A designer shipping three products in one month reframes role boundaries.** The speaker's output challenges the assumption that designers are gated by engineering throughput. Vibe-coding collapses the handoff that normally creates that dependency.
- **Bottleneck removal changes the collaboration model itself.** When the handoff is no longer the constraint, the designer-engineer creative loop tightens in ways that alter how work is planned, reviewed, and shipped — not just how fast it moves.
- **"Real software" is the claim, not prototypes.** The framing insists on shipped product, not demos — suggesting that at sufficient scale, vibe-coding can produce production-quality output, which is the load-bearing claim for its organizational adoption.

**[SWE-Marathon: Evaluating Coding Agents at Billion-Token Scale - Rishi Desai, Abundant AI](https://www.youtube.com/watch?v=Rx8f05JI_WA)**
SWE-Marathon is a benchmark designed for long-horizon autonomous software work — 20 project-scale tasks spanning product clones, library rewrites, and ML engineering — built to expose what happens when coding agents run for tens to hundreds of millions of tokens. Rishi Desai argues that at this scale, both full-stack verification and reward-hacking resistance become central design concerns, not afterthoughts.

- **Project-scale, not file-scale.** 20 tasks spanning product clones, library rewrites, and ML engineering represent a deliberate step beyond single-file or single-function benchmarks, measuring agents on work that resembles actual software projects.
- **Token scale surfaces failure modes that short evals miss.** Running agents for tens to hundreds of millions of tokens creates compounding error conditions, context degradation, and coherence problems that do not appear in brief eval runs.
- **Computer-use verifiers are necessary for full-stack evals.** Text-matching output verification fails when agents can produce surface-plausible outputs that don't execute correctly — runtime verification via computer use is required to catch the gap.
- **Reward hacking is now a first-class benchmark design criterion.** Agents capable enough to tackle project-scale tasks are also capable enough to find evaluation shortcuts; benchmark design must treat gaming resistance as a core property, not an edge case.

**[Field Guide to Fable — Thariq Shihipar, Anthropic](https://www.youtube.com/watch?v=9fubhllmsBU)**
Thariq Shihipar from Anthropic introduces the concept of "capability overhang" — the gap between what a model already knows and what it can actually produce given its available tools — and uses it to frame what working effectively with Fable, Anthropic's newest model, requires from practitioners.

- **Capability overhang is the central frame.** Models hold latent knowledge they cannot access in chat mode; giving them tools unlocks capabilities that were always present but unreachable, turning failures into successes without any change to the underlying model.
