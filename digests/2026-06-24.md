## AWS What's New

**[AWS HealthOmics now supports Nextflow profiles](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-healthomics-nextflow-profiles/)**
HealthOmics customers can now activate predefined Nextflow execution settings at run time instead of editing workflow source code. Lets teams switch between execution configurations without modifying the underlying workflow, relevant to HIPAA-eligible bioinformatics pipelines.

**[AWS introduces Lambda MicroVMs for isolated execution of user and AI-generated code](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-lambda-microvms/)**
A new serverless compute primitive giving each user or job its own VM-isolated environment with near-instant launch/resume and state preservation. Targets multi-tenant apps that need to run user- or AI-generated code without managing virtualization infrastructure.

**[AWS Transform for migrations now supports all AWS commercial regions as migration targets](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-transform-migrations-region-expansion/)**
Migrated resources (landing zones, network infra, rehosted servers) can now be deployed to any AWS commercial region, including new targets like US East (N. California), Africa (Cape Town), Asia Pacific (Bangkok), and Asia Pacific (Hong Kong). Eases data residency compliance for migrations.

**[AWS Network Firewall updates default drop action for improved connection reliability](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-network-firewall-updates-default-drop-action)**
New firewall policies now default to "Application drop established (server-directed only)" instead of the prior bidirectional default. No action required to benefit; applies only to newly created policies.

**[AWS Batch now supports customer-ordered instance allocation strategies](https://aws.amazon.com/about-aws/whats-new/2026/06/batch-ordered-allocation-strategies/)**
Two new allocation strategies — Best Fit Progressive Ordered (BFPO) and Spot Capacity Optimized Prioritized (SCOP) — let users manually define instance type ordering for on-demand and Spot compute environments based on workload performance characteristics.

**[AWS IAM Identity Center now supports separate quotas for AWS accounts and applications](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-identity-center-separate-quotas/)**
Account and application quotas are now independent, each defaulting to up to 7,000, so consuming one no longer eats into the other's capacity. Quota increases are available via AWS Service Quotas; customers with existing higher limits are auto-grandfathered.

**[Amazon MSK now offers AI Agent Skills to help developers operate MSK efficiently and accelerate migrations to MSK](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-msk-ai-agent-skills)**
New AI Agent Skills give AI coding assistants expert guidance for MSK troubleshooting, sizing, configuring, monitoring, and Kafka-to-MSK migrations, including migrating to MSK Express for up to 3x more throughput per broker.
