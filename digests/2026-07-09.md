## AWS What's New

**[Amazon GameLift Streams introduces secure terminal access for stream sessions](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-gamelift-streams-terminal-access/)**
Amazon GameLift Streams now provides a Stream Session Admin Shell — a secure terminal connection into the live runtime environment of an active stream session for real-time troubleshooting. It gives access to logs, running processes, GPU utilization, and application state without requiring SSH keys, open ports, or infrastructure credentials.

**[Amazon S3 Vectors is now available in AWS GovCloud (US) Regions](https://aws.amazon.com/about-aws/whats-new/2026/07/s3-vectors-available-aws-govcloud-regions/)**
Amazon S3 Vectors — purpose-built vector storage designed for AI agents, RAG, and semantic search at billion-vector scale — is now available in both AWS GovCloud (US-East) and GovCloud (US-West). It provides dedicated APIs for storing, accessing, and querying vectors without provisioning infrastructure, matching S3's elasticity and durability model.

**[AWS Security Hub extends unified security management to Microsoft Azure](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-security-hub-supports-monitoring-microsoft-azure/)**
AWS Security Hub now monitors Microsoft Azure resources alongside AWS, providing a single interface for risk analytics, cloud security posture management, vulnerability management, and security response across both clouds. Previously, customers running workloads in both environments had to operate separate security tooling for each, making holistic risk prioritization difficult.

**[Amazon ECS Managed Instances reduces GPU management fees by up to 60%](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-ecs-managed-instances-gpu-price/)**
Effective July 1, 2026, ECS Managed Instances management fees for GPU and accelerated instance types are automatically reduced: G-series instances see a 35% reduction, while P-series and AWS Trainium instances see a 60% reduction. No action is required from customers already using these instance types with ECS Managed Instances.

**[Amazon EMR Serverless now supports larger worker sizes to run more compute and memory intensive workloads](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-emr-serverless/)**
EMR Serverless now supports worker configurations of up to 32 vCPUs with 244 GB of memory — doubling the previous ceiling of 16 vCPUs and 120 GB. For shuffle-heavy workloads, larger workers reduce inter-executor data transfer; for memory-intensive jobs, they improve both runtime performance and cost profiles.

**[Amazon EC2 C8ine instances are now available in AWS Europe (Frankfurt) region](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-ec2-c8ine-aws-frankfurt/)**
EC2 C8ine instances — powered by custom sixth-generation Intel Xeon Scalable processors on the latest AWS Nitro cards — are now available in the Frankfurt region, delivering up to 43% higher performance than prior-generation C6in instances. They also offer up to 2.5x higher packet performance per vCPU compared to the previous generation.

**[Amazon SageMaker now supports data lineage in IAM-based domains](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-sagemaker-data-lineage-iam-domain)**
SageMaker Unified Studio now supports OpenLineage-compatible data lineage in IAM-based domains, capturing lineage events from Apache Spark jobs on Amazon EMR, AWS Glue, SageMaker Visual ETL, and notebooks — extending a capability previously available only in IAM Identity Center-based domains. The interactive lineage graph supports configurable depth and event timestamp mode down to column-level lineage.

**[Amazon Redshift RG instances now available in AWS GovCloud (US) Regions](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-redshift-rg-instances-aws-govcloud)**
Redshift RG instances, powered by AWS Graviton processors, are now available in GovCloud (US-West) and GovCloud (US-East), delivering up to 2.4x faster performance than prior RA3 instances at 30% lower price per vCPU. They include a custom vectorized data lake query engine with support for Apache Iceberg and Parquet workloads run directly on cluster nodes.

**[Amazon RDS for Oracle now supports Oracle Database 26ai](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-rds-oracle-database-26ai/)**
Amazon RDS for Oracle now supports Oracle Database 26ai, Oracle's latest Long Term Support Release, with Amazon Bedrock integration providing access to foundation models including Anthropic Claude, Amazon Nova, and Meta Llama. The release enables the Select AI feature for natural-language SQL generation and supports implementing RAG directly from within the database.

**[Amazon S3 Express One Zone is now available in the AWS Europe (Frankfurt) Region](https://aws.amazon.com/about-aws/whats-new/2026/07/s3-express-one-zone-europe-frankfurt/)**
The S3 Express One Zone storage class — designed for consistent single-digit millisecond data access for frequently accessed and latency-sensitive workloads — is now available in the Frankfurt region. It delivers data access up to 10x faster and request costs up to 80% lower than S3 Standard.

**[Your coding agent doesn't always follow your rules — Talha Sheikh, Checkout.com](https://www.youtube.com/watch?v=MpZzWMdmQCE)**
The core problem Talha Sheikh addresses at Checkout.com: coding agents deviate from specified rules in production, and relying on the model to self-enforce constraints does not hold. An agent harness that intercepts and applies rules at runtime — on every invocation — is the proposed fix.

- **Rule-following is unreliable by design in current agents.** The talk opens from the premise that deviation is the expected behavior, not an edge case — a framing that shifts responsibility for compliance from the model to the surrounding infrastructure.
- **A harness enforces constraints in real time, every time.** Rather than prompting more compliant behavior, the proposed approach moves rule enforcement to the harness layer, where it operates consistently regardless of model drift.
- **Compliance becomes an infrastructure property, not a model property.** Separating governance from capability — the harness handles the former, the model handles the latter — makes the system auditable and enforceable in ways that prompt-level controls are not.

**[Your LLM Deception Monitor Is Broken. The Fix Is in the Training Data - Sachin Kumar, LexisNexis](https://www.youtube.com/watch?v=IQkVMvXQKLY)**
Fine-tuning can write backdoors into LLMs that pass all behavioral evals and monitors while remaining dormant until a specific trigger is encountered — one the test suite never exercises. Sachin Kumar argues that runtime behavioral monitoring cannot reach this class of vulnerability, and that the correct diagnostic approach operates on what the training data changed in the model's activation space.

- **Green evals do not rule out backdoors.** A model can pass every behavioral test while carrying a sleeper-agent trigger, because the backdoor only fires on an input the test suite never hits. Behavioral testing cannot see what it does not probe.
- **Crosscoders dilute the signal you need.** Joint cross-model interpretability features average out the directional activation shift that a backdoor creates, pushing it to the noise floor. The standard interpretability tool is counterproductive for detecting this threat.
- **A backdoor is a directional shift in activation space.** Framing it this way opens a diagnostic path: look at what fine-tuning changed in the model's activations rather than what the model produces in response to test prompts.
- **The fix is upstream — in the training data.** Because the backdoor was written during fine-tuning, its artifact lives in the weights and its evidence is in the training data, not in runtime outputs. The monitoring layer is checking the wrong place.

**[Beyond the Harness: A Journey Towards Adaptative Engineering - Rajiv Chandegra, Annicha Labs](https://www.youtube.com/watch?v=qdZzND79mcg)**
Rajiv Chandegra argues that as product-building becomes commoditized, the remaining engineering value lies in complex, real-world problems — and that those problems expose a structural liability in today's AI harnesses: they impose predetermined structures on problems that are dynamic and do not hold still.

- **Product building is commoditized; hard problem-solving is not.** The opening claim repositions engineering value away from shipping products and toward the messy, tangled challenges of the physical and social world that AI cannot yet structure for itself.
- **Fixed harnesses are a liability in shifting systems.** A harness that imposes a predetermined structure on a problem is brittle when the problem changes shape — and real-world challenges change continuously in ways that static scaffolding cannot anticipate.
- **Adaptive harnesses — responsive in real time — are the prescription.** The analogy to "a great leader" (the snippet cuts off here) frames the model as responsiveness and situational judgment, not static rule application.
- **The opportunity scales with model capability.** As models grow more capable, the well-structured, easily defined problems are automated away; what remains for engineers is precisely the work that resists clean structuring — which harness rigidity is least equipped to handle.

**[How we taught agents to use good retrieval - Hanna Lichtenberg, Mixedbread AI](https://www.youtube.com/watch?v=1IdzkRVmWAA)**
Hanna Lichtenberg pushes back on the recurring "RAG is dead, just use grep" narrative by grounding the argument in real knowledge tasks — legal research, financial analysis, Deep Research — where semantic search has real failure cases but keyword search also falls short. The Oracle Gap, the performance delta between perfect retrieval and what systems actually achieve, frames the true problem.

- **"RAG is dead" is a recurring cycle, not a signal.** The talk names the pattern explicitly: a new version of the same claim surfaces regularly, which suggests it is a rhetorical habit rather than an empirical conclusion supported by the hard use cases.
- **Both semantic search and keyword search have genuine failure modes.** Acknowledging semantic search's weaknesses while refusing to treat BM25 or grep as substitutes is the load-bearing claim: the debate is about which technique gets closer to perfect, not which one is correct.
- **The Oracle Gap defines the real problem.** The performance difference between perfect retrieval and what any real system delivers reframes the question from "which technique?" to "how close can we get?" — a more tractable and honest engineering objective.
- **High-stakes knowledge tasks set the standard.** Deep Research, financial analysis, and legal research are chosen as the evidence base — domains where retrieval failure is costly, making them a harder and more meaningful test than toy benchmarks.

**[What if the harness mattered more than the model? - Aditya Bhargava, Etsy](https://www.youtube.com/watch?v=2e9ANoOEn28)**
Aditya Bhargava challenges the "simple harness + tools + loop" framing that has followed rapid model capability gains, arguing it embeds a structural choice: outsource capability to the model and accept dependence on large, proprietary systems. The alternative he proposes is investing in harnesses capable of elevating models that can be run locally.

- **"Simple harness + loop" is a direction, not just a simplification.** Bhargava identifies minimal-harness thinking as embedding a long-term structural bet — that capability gains will keep coming from model scale — rather than treating it as a neutral engineering preference.
- **Over-reliance on large proprietary models is the downstream risk.** If harnesses remain thin, capability must come from model scale, which concentrates leverage with a small number of vendors and reduces the optionality available to the teams using them.
- **Harnesses that elevate local models are the counter-proposal.** The alternative is not waiting for better open models but building better scaffolding that lifts the performance of models that can already be run without vendor dependency.
- **The talk is a bet on harness investment as the strategic variable.** For practitioners at organizations like Etsy, where you concentrate engineering effort — on model access or on harness sophistication — determines what you can own and operate independently.
