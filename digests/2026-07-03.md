## The Pragmatic Engineer

**[How Kent Beck shapes the software engineering industry](https://www.youtube.com/watch?v=ddHQQtjIOpw)**
Kent Beck — creator of Extreme Programming, pioneer of TDD, co-creator of JUnit, and co-author of the Agile Manifesto — joins the Pragmatic Engineer podcast to re-examine foundational software ideas in the age of AI. His central provocation: the industry is accumulating new code far faster than it is building the trust required to rely on it.

- **Trust deficit, not a code deficit.** Beck's stated position is that we are failing to accumulate trust at the same rate as new code is being generated — a gap that, unlike traditional backlogs, is invisible in most delivery metrics.
- **A career spent building feedback loops.** XP, TDD, and JUnit were each mechanisms for creating trust at the moment of writing; the episode revisits whether those loops are still adequate when AI generates code in bulk without the same deliberate feedback.
- **Re-examining the foundations.** Beck is explicitly re-examining his ideas rather than extending them — signaling that adapting prior principles may not be enough and that the AI era may require conceptually new answers.

## Daniel Pink

**[You're Not That Special](https://www.youtube.com/watch?v=9NzeLQYctLA)**
Daniel Pink argues that harsh self-criticism is not humility — it is narcissism running in the opposite direction. Treating a personal failure as a data point rather than a verdict is what makes it actually useful.

- **Self-criticism and narcissism share the same root.** The harshest self-critics are still centering everything on themselves; they have simply flipped the valence. The ego is the common factor, not the sign.
- **Reframe failure as data, not as verdict.** A moment where things went sideways is a single data point. Elevating it to a judgment about your identity converts something correctable into something paralyzing.
- **Universality is the unlock.** Pink's anchor — "you're a human who had a moment" — invokes the shared human experience and removes the exceptionalism that makes self-criticism useless in either direction.

## AWS What's New

**[Amazon Bedrock AgentCore increases default runtime quota limits](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-bedrock-agentcore-increases-default-runtime-quota-limits/)**
Amazon Bedrock AgentCore has raised its default runtime quotas to support larger agent deployments. US East (N. Virginia) and US West (Oregon) now allow up to 5,000 active concurrent sessions; all other supported regions support 2,500. Every supported region now handles 200 agent interactions per second and 25 new sessions per second.

**[Amazon CloudWatch supports creating alarms from log queries](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-cloudwatch-log-alarms/)**
CloudWatch now lets you create alarms directly on log query results, eliminating the need to first create metric filters or custom metrics as intermediate steps. You set the threshold directly on the query, streamlining the path from log analysis to active alerting without leaving the log workflow.

**[ECS Service Connect now supports Zone-Aware routing](https://aws.amazon.com/about-aws/whats-new/2026/07/ecs-service-connect-zone-aware/)**
ECS Service Connect can now preferentially route service-to-service traffic to endpoints in the same Availability Zone as the originating task, reducing cross-AZ data transfer costs and latency. Traffic weights adjust dynamically as endpoints scale to maintain balanced load across target services.

**[Amazon ECS now provides real-time deployment observability in the AWS Management Console](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-ecs-aws-management-console/)**
Amazon ECS now surfaces a live deployment timeline in the console, letting teams track progress, monitor health, and diagnose failures in real time without switching between log streams and event lists. The feature surfaces issues as they occur, reducing the time needed to troubleshoot and resolve deployment failures.

**[AWS Artifact now includes Assurance Assistant for compliance inquiries](https://aws.amazon.com/about-aws/whats-new/2026/07/aws-artifact-assurance-assistant/)**
AWS Artifact's new Assurance Assistant is an AI-powered capability that generates citation-backed answers to security and compliance questions about AWS services. It targets compliance officers, third-party risk managers, and auditors to accelerate vendor assessments and due diligence questionnaire completion.

**[AWS Partner Central now supports AWS Marketplace listings for co-selling](https://aws.amazon.com/about-aws/whats-new/2026/07/aws-marketplace-co-selling-support/)**
Partners can now associate existing AWS Marketplace product listings directly with co-sell opportunities in AWS Partner Central, eliminating the need to maintain separate solution catalogs for each workflow. Previously, co-selling required creating dedicated solutions distinct from marketplace catalog entries.

**[Amazon RDS announces Cross-Region Automated Backups in four additional AWS Regions](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-rds-cross-region-automated-backups-additional-aws-regions/)**
Cross-Region Automated Backup replication for Amazon RDS now covers four more regions: Mexico (Central), Asia Pacific (Taipei), Asia Pacific (New Zealand), and Asia Pacific (Thailand), each with specific supported destination regions. The expansion removes the need for manual off-site backup configuration for customers in those geographies.

**[Amazon Bedrock AgentCore now available in four additional AWS Regions](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-bedrock-agentcore-four-additional-regions/)**
Amazon Bedrock AgentCore expands to Asia Pacific (Bangkok), Asia Pacific (Malaysia), Europe (Milan), and Europe (Spain). Customers in these regions can now build, connect, and optimize AI agents on the platform with infrastructure-level security enforcement.

**[Amazon ECS now supports configurable deployment circuit breaker settings](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-ecs-circuit-breaker-settings/)**
ECS deployment circuit breakers — which automatically detect failed deployments and roll back to the last stable version — can now be customized per service. Teams can tune thresholds to match application startup behavior, deployment needs, and acceptable failure tolerance instead of relying on a single shared setting.

**[Amazon GuardDuty adds sensitive file modification threat detections](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-guardduty-sfm/)**
GuardDuty Runtime Monitoring adds three new threat detections that fire when sensitive files are modified on EC2 instances or container workloads running on EKS or ECS. The findings target post-compromise attacker activity by monitoring critical system files, authentication settings, and system logs.

## AWS Blog

**[Upgrade Amazon EKS clusters with confidence using Kubernetes version rollbacks](https://aws.amazon.com/blogs/aws/upgrade-amazon-eks-clusters-with-confidence-using-kubernetes-version-rollbacks/)**
