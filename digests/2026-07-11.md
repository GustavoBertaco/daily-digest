## AWS What's New

**[OAuth support for the AWS MCP Server](https://aws.amazon.com/about-aws/whats-new/2026/07/oauth-aws-mcp-server/)**
AI agents can now connect directly to the AWS MCP Server using industry-standard OAuth via AWS Sign-In, with no additional authentication software required. Existing IAM identities, permissions, and governance controls apply automatically; developers can authorize agents interactively via browser or programmatically through headless (non-interactive) authorization.

**[Amazon Timestream for InfluxDB now publishes database state change events to Amazon EventBridge](https://aws.amazon.com/about-aws/whats-new/2026/07/timestream-influxdb-eventbridge/)**
Timestream for InfluxDB now emits lifecycle events to EventBridge covering creation, deletion, compute and storage scaling, parameter group updates, maintenance windows, reboot, and both successful and failed outcomes. Customers can use EventBridge rules to react programmatically to database operations without polling the API for status.

**[AWS Client VPN extends availability to four additional AWS Regions](https://aws.amazon.com/about-aws/whats-new/2026/07/aws-client-vpn-four-additional-regions/)**
AWS Client VPN is now available in Canada West (Calgary), Mexico (Central), Asia Pacific (New Zealand), and Asia Pacific (Taipei). The fully managed, pay-as-you-go service enables secure remote-workforce connectivity to AWS or on-premises resources without hardware VPN appliances or complex operational management.

**[Amazon SageMaker Unified Studio adds custom asset types to the catalog in IAM-based domains](https://aws.amazon.com/about-aws/whats-new/2026/07/smus-custom-asset-types-iam/)**
Domain administrators can now catalog any asset format in SageMaker Unified Studio for IAM-based domains — examples include medical imaging files in S3, PowerBI revenue dashboards, or third-party PDF research reports. Custom asset types bring all assets, regardless of format, into the SageMaker catalog so teams can search, discover, and subscribe to them centrally.

**[Amazon SageMaker HyperPod now supports deep health checks for Slurm clusters with continuous provisioning](https://aws.amazon.com/about-aws/whats-new/2026/07/deep-health-check-continuous-slurm/)**
HyperPod adds proactive GPU accelerator health checks for Slurm-orchestrated clusters created with continuous provisioning, enabling hardware validation as new instances come online. This pairs the flexibility of asynchronous, non-blocking instance scaling with comprehensive health verification at provisioning time.

**[Amazon SageMaker Feature Store now supports batch feature writes and record listing](https://aws.amazon.com/about-aws/whats-new/2026/07/amzn-sgm-feature-store-batch-write-list/)**
Feature Store gains two new capabilities: `BatchWriteRecord` allows writing multiple records across multiple feature groups in a single request for high-throughput ingestion, and a new listing API lets data scientists discover stored records without needing to know each record identifier in advance.

**[Amazon MSK Replicator now supports replication from external Apache Kafka clusters to MSK Standard brokers](https://aws.amazon.com/about-aws/whats-new/2026/07/amazon-msk-replicator-external-kafka-standard-broker-support)**
MSK Replicator can now replicate data from external Kafka clusters — on-premises, self-managed on AWS, or other cloud providers — to MSK Standard brokers, extending existing support beyond MSK Express brokers. This enables workload migrations to MSK Standard, disaster recovery failover, and cross-environment data distribution scenarios.

**[AWS Config now supports 191 additional managed rules](https://aws.amazon.com/about-aws/whats-new/2026/07/aws-config-additional-managed-rules)**
AWS Config adds 191 new managed rules across Amazon Bedrock, SageMaker, ECS, EKS, RDS, Redshift, S3, and CloudTrail. New rules target encryption, logging, public access, network security, and data protection — expanding built-in governance coverage to AI workloads alongside core cloud infrastructure.

## AI Engineer

**[The Golden Age of AI Engineering — Alexander Embiricos & Romain Huet & Peter Steinberger, OpenAI](https://www.youtube.com/watch?v=pMggiOb18tc)**
OpenAI's talk at AI Engineer uses two consecutive Dev Day demos — one year apart — to make the pace of AI engineering progress concrete: in 2024, the model couldn't run or verify its own code, so the presenter had to trust it blindly live on stage; in 2025, the same class of demo controlled a full camera and lighting rig autonomously because the model could now test its own work. The session treats this gap as the defining signal for where agentic coding tooling is heading.

- **Self-verification is the capability threshold that changes the risk profile.** When a model can test its own outputs, the category of "agentic code generation" shifts from faith-based to verifiable. The 2024-to-2025 jump in the demo framing illustrates this cleanly.
- **Codex model release cadence compressed from ~15 months to ~6 weeks.** That pace means assumptions baked into architecture or integration choices can go stale before a project ships — teams need to think about building on behavioral interfaces, not specific model snapshots.
- **Shared infrastructure with the responses API and an open-source harness.** Codex is built on the same foundation as other OpenAI tooling, so mental models and integration patterns already established for the responses API carry over directly.
