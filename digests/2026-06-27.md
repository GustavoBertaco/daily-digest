## AI Engineer

- **Production agents need real observability.** Feedback loops, evals, and monitoring/observability are treated as first-class concerns, not afterthoughts.
- **Long context and tooling are separate problems.** Long-context handling and the design of tools/skills are called out as distinct challenges from the core loop itself.
- **A2A and enterprise contribution model matter at scale.** Supporting agent-to-agent (A2A) protocol and an enterprise contribution model are framed as necessary once thousands of government bodies are involved.

**[A Genius With Amnesia - Victor Savkin, Nx](https://www.youtube.com/watch?v=jVjt-2g8NMY)**
Victor Savkin's thesis, via an analogy: today's coding agents are like a world-class engineer who can only see a tiny corner of the codebase and forgets everything between interactions — skill alone doesn't compensate for that structural blindness.

- **Context, not capability, is the bottleneck.** The argument is that raw model skill ("John Carmack in his prime") doesn't translate into value if the agent's view of the codebase is too narrow.
- **Memory loss between interactions is the core failure mode.** Forgetting state session to session is presented as the central problem to fix, not a minor inconvenience.
- **The framing is a call to action.** Savkin positions this as something the industry "needs to fix," tying it to his work on Nx (agentic monorepo platform) and Polygraph (meta-harness for agent autonomy).

**[The Log Is The Agent - Ishaan Sehgal, Omnara](https://www.youtube.com/watch?v=UPwGaM2MKHY)**
Ishaan Sehgal's thesis, via a video-game analogy: an agent's true identity isn't the engine, the compute, or the tools — it's the persisted data/save file, since everything else is replaceable without losing the agent's actual state.

- **Identity lives in the data, not the infrastructure.** Swapping the underlying compute ("PlayStation") doesn't kill the agent as long as the save file survives.
- **Engine, compute, and tools are all ruled out explicitly.** The analogy walks through and rejects the game loop, the console, and the controller in turn before landing on the save file as the real seat of identity.
- **This reframes what to architect for durability.** If the log/save-state is the real asset, persistence and portability of that data becomes the priority over the runtime around it.

**[Recursive Coding Agents - Raymond Weitekamp, OpenProse](https://www.youtube.com/watch?v=3hXJI2q0Jz8)**
Raymond Weitekamp explores how Recursive Language Models and their inference-time compute patterns map onto coding agents to improve performance and reliability.

- **Inference-time compute patterns transfer to coding agents.** The talk's core move is applying recursive-LM techniques, originally framed at the model level, to agent-level coding workflows.
- **Recursion is positioned as a reliability lever.** The framing ties recursive structure specifically to performance and reliability gains, rather than treating it as just an efficiency trick.

**[Production Evals For Agentic AI Systems - Nishant Gupta, Meta Superintelligence Labs](https://www.youtube.com/watch?v=vljxQZfJ9wY)**
Nishant Gupta's thesis: as systems move from chat interfaces to autonomous agents that reason, plan, and use tools, traditional offline benchmarks and static-dataset evaluation break down and can't capture real-world complexity, non-determinism, or operational risk.

- **Offline benchmarks don't generalize to agents.** Static datasets miss the non-determinism inherent to agentic systems operating in production.
- **Evaluation has to become an architectural concern.** The talk frames eval system design as needing its own patterns, not a bolt-on to existing model-eval practice.
- **Operational risk is part of the eval surface.** Real-world deployment risk is explicitly called out alongside correctness as something eval systems must capture.

**[Build Systems, Not Code - Angie Jones, Agentic AI Foundation](https://www.youtube.com/watch?v=ZD9-4fW2HhM)**
Angie Jones' thesis: as AI coding agents take over more line-by-line code writing, engineering craftsmanship doesn't disappear — it moves up a layer, from writing code to designing the agentic systems around it.

- **The craftsmanship question is the open one.** She frames the talk around whether agents writing the code means engineers lose the joy of building, before arguing they don't.
- **Craftsmanship relocates, it doesn't vanish.** The engineering flow state she describes comes from designing systems, not from typing every line personally.
- **Classic engineering principles still apply.** Systems thinking, decomposition, and separation of concerns are named explicitly as the skills that carry over into agentic system design.

**[The Miranda Hypothesis: How Hamilton Poisoned Persona Evals - Jacob E. Thomas, Results Gen](https://www.youtube.com/watch?v=IJXjTLPzvAU)**
Jacob E. Thomas's thesis: persona-eval pipelines for character-based AI systems are structurally blind to their dominant failure mode, because the evals (LLM-as-judge, personality-scale benchmarks, behavioral consistency scores) are built to detect "convincingness" — and convincingness is exactly what the failure produces.

- **The named failure mode is "the Miranda Hypothesis."** Illustrated with an Alexander Hamilton persona simulation scoring well on personality fidelity while actually sounding like it learned the character from the Broadway musical rather than the historical figure.
- **High eval scores can mask the wrong kind of fidelity.** A persona system can be rated highly convincing while being convincingly *wrong* — faithful to a popularized version of a character rather than the real one.
- **Today's eval methods can't see this by design.** Because LLM-as-judge and consistency-based benchmarks optimize for convincingness, they're structurally unable to flag a failure mode that *is* convincingness.
