## Daniel Pink

**[4 Habits To Reduce Negative Thinking!](https://www.youtube.com/watch?v=p-T5ldqJPEU)**
Yale psychologist Susan Nolen-Hoeksema's decades of research on rumination provide the scientific backbone for this video, which packages the findings into four concrete habits for reducing negative thinking.

- **Rumination is the stronger predictor.** Nolen-Hoeksema's research identified rumination as one of the strongest predictors of anxiety and depression — and found it is often a bigger factor than the event that originally triggered the negative thinking.
- **The target is the cycle, not the cause.** Because rumination amplifies distress beyond the original event, intervening in the rumination loop itself offers leverage that addressing root causes alone may not.
- **Habits as the delivery mechanism.** Framing the output as four learnable habits provides a behavioral, actionable entry point grounded in decades of academic research.

## AWS What's New

**[Amazon CloudWatch Logs enriches log events with AWS resource tags](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-cloudwatch-logs-resource-tags/)**
CloudWatch Logs now adds resource tags — such as team ownership, environment, cost center, or application name — directly to log events at ingestion time, requiring no changes to logging instrumentation. Tags are immediately usable in log queries for filtering and scoping analysis without having to build custom pipelines.

**[AWS CloudFormation and CDK accelerate development feedback loops with pre-deployment validation on all stack operations](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-cloudformation/)**
CloudFormation now runs pre-deployment validation on Create Stack and Update Stack operations, catching common deployment errors in seconds before any resource provisioning begins. This eliminates waiting through a full provision-and-rollback cycle to discover preventable failures, accelerating manual iteration, CI/CD pipelines, and AI agent provisioning workflows alike.

**[AWS CloudFormation and CDK express mode speeds up infrastructure deployments by up to 4x](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-cloudformation-cdk/)**
Express mode reduces deployment time by up to 4x (per internal benchmarks) by completing stack operations when CloudFormation confirms resource configuration is applied, skipping extended stabilization checks such as traffic readiness, region propagation, and resource cleanup. This enables faster iteration cycles for both developers and AI agents building infrastructure.

**[Amazon RDS Enhances IAM Database Authentication with Connection Rate Scaling](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-rds-iam/)**
RDS IAM database authentication now scales dynamically with available instance resources rather than hitting a fixed ceiling, enabling enterprise workloads with high-volume connection patterns to use IAM auth at scale. The number of new IAM authentication requests an instance can handle is now determined by available resources and workload characteristics.

**[AWS Parallel Computing Service supports in-place Slurm major version upgrades](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-parallel-computing-service-upgrade/)**
AWS PCS now supports managed in-place Slurm version upgrades of up to three major versions ahead, with no disruption to running jobs. The upgrade — triggered by updating the Cluster configuration via Console, CLI, or UpdateCluster API — covers all managed Slurm components, including the controller, accounting database, and REST API.

**[Amazon ElastiCache T4g nodes now available in additional AWS Regions](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-elasticache-t4g-additional-aws-regions/)**
T4g nodes, powered by AWS Graviton2 processors with a burstable CPU baseline, are now available in Africa (Cape Town), Asia Pacific (Jakarta), Asia Pacific (Osaka), AWS GovCloud (US-East), and AWS GovCloud (US-West). They are suited for applications with temporary usage spikes that need burst CPU capacity without paying for consistently high compute.

**[Amazon Neptune announces dual stack support with IPv6](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-neptune-ipv6/)**
Neptune clusters can now accept connections over IPv4, IPv6, or both simultaneously via dual-stack mode, allowing organizations to adopt IPv6 while preserving backward compatibility with existing IPv4 deployments. Two configurations are available: private dual-stack (IPv6 endpoints isolated from the internet) and public dual-stack (IPv6 endpoints accessible externally).

**[AWS End User Messaging RCS now supports rich media and interactive messaging](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-end-user-messaging-rcs/)**
The new SendRcsMessage API enables sending rich cards, carousels, images, videos, and interactive suggestion buttons to recipients across all 22 supported RCS countries. Recipients can confirm appointments, browse product catalogs, complete payments in a webview, share their location, or interact with an AI agent — all without leaving their messaging app.

**[Amazon Time Sync Service adds support for Microsecond accurate time on 26 additional EC2 instance types in all commercial regions](https://aws.amazon.com/about-aws/whats-new/2026/06/ec2-time-sync-precision-time-placement-group)**
Microsecond-accurate time and nanosecond-precision hardware timestamps — built on the Nitro System's reference clocks — are now available on 26 additional EC2 instance types across all commercial regions. This lets applications order events precisely, measure one-way network latency, and increase distributed transaction speed.

**[Announcing general availability of Amazon WorkSpaces for AI agents](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-workspaces-ai/)**
WorkSpaces for agents is now generally available, providing AI agents secure access to desktop applications — ERP systems, CRMs, mainframes, and proprietary tools — through managed cloud WorkSpaces environments. The target scenario is enterprises whose critical business processes run on desktop applications that are too costly to modernize but need to be automated.
