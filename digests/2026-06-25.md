## AWS What's New

**[Amazon CloudWatch now supports tags on dashboards](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-cloudwatch-tags-on-dashboards)**
CloudWatch dashboards now support tagging for organization, categorization, and access control. The `PutDashboard` API accepts an optional `Tags` parameter (up to 50 tags per dashboard), and `TagResource`, `UntagResource`, and `ListTagsForResource` now support dashboard ARNs.

**[Amazon CloudWatch Logs supports managed syslog ingestion](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-cloudwatch-syslog-ingestion/)**
CloudWatch Logs can now ingest syslog messages directly from firewalls, routers, switches, and Linux servers over TCP, TCP+TLS, or UDP to a VPC endpoint, with no agent installation required. Supports RFC 5424, RFC 3164, and Cisco FTD/ASA syslog formats.

**[SageMaker Notebook Instances now support G6e instance types](https://aws.amazon.com/about-aws/whats-new/2026/03/g6e-new-launch-sagemaker-notebook-instances/)**
SageMaker notebook instances now support EC2 G6e instances (up to 8 NVIDIA L40S GPUs, 48 GB memory per GPU, 3rd-gen AMD EPYC CPUs), delivering up to 2.5x better performance than G5 for interactive model testing, deployment, and generative AI fine-tuning.

**[Amazon Bedrock AgentCore Memory now supports cross-account access](https://aws.amazon.com/about-aws/whats-new/2026/06/agentcore-memory-cross-account-access)**
AgentCore Memory now supports multi-account architectures: principals in one account can call memory data-plane APIs against resources in another via resource-based policies, and memory delivery destinations (S3, SNS, Kinesis Data Streams) can reside in a separate account.

**[AWS HealthOmics now supports ephemeral storage for private workflows](https://aws.amazon.com/about-aws/whats-new/2026/06/healthomics-scratch-storage/)**
HealthOmics private workflow tasks now get a dedicated local scratch volume mounted at `/tmp`, improving run-time consistency and cost for scratch-heavy bioinformatics workloads like genomic sequence alignment, BAM sorting, and variant calling.

**[Amazon Cognito now supports customer managed key for encryption at rest](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-cognito-customer-managed-key)**
Cognito user pool data at rest can now be encrypted with customer managed KMS keys instead of only AWS-owned keys, giving customers direct control to define governance policies and revoke access by disabling or deleting their key.

**[Automated Reasoning checks in Amazon Bedrock Guardrails add new policy refinement workflows](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-bedrock-guardrails/)**
Bedrock Guardrails' Automated Reasoning checks — which use formal logic to mathematically validate generative AI responses against a defined policy — gain new automated refinement workflows to help customers improve policy quality with less manual effort, since validation accuracy depends heavily on how well the policy is defined.

**[Amazon CloudWatch launches OTel Container Insights for Amazon EKS](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-cloudwatch-otel-amazon-eks/)**
New CloudWatch OTel Container Insights for EKS collects infrastructure metrics at 30-second granularity via open-source receivers (cAdvisor, Kube State Metrics, NVIDIA DCGM), tagged with OpenTelemetry semantic conventions and Kubernetes labels for cross-node/pod/workload correlation in a single PromQL query, plus pre-built dashboards for cluster, node, and pod visibility.
