# 🤖 AI & Machine Learning

## AI Engineer

**[The Missing Layer After Launch - Raphael Kalandadze, Wandero AI](https://www.youtube.com/watch?v=kZsf_Sfm7RU)**
Running agents in production exposes a structural observability gap: when an agent fails a customer — a dropped constraint, a stale price, a confident wrong answer — nothing crashes and no log fires. The failure lives in the conversation, not the stack trace. Wandero AI's response was to build an operations layer staffed entirely by agents.

- **Agent failures are silent by default.** Unlike conventional software failures, a broken agent produces no error signal in the infrastructure layer — the damage surfaces only in the conversation record, making standard monitoring insufficient.
- **Agents monitoring agents.** Rather than a human-staffed ops team, the team deployed a dedicated monitoring agent that continuously judges production conversations to detect where the product actually let a customer down.
- **A distinct operational discipline.** Operating an agent product is fundamentally unlike operating software — the error surface is conversational and contextual, not infrastructural, requiring purpose-built tooling and new mental models.

**[Continual Learning for AI Agents: From Failures to Durable Improvements - Soheil Feizi, RELAI](https://www.youtube.com/watch?v=2IxD9OB3XuQ)**
Static benchmarks cannot fully capture how agents fail in production. Soheil Feizi of RELAI introduces verifiable continual learning — a framework for converting production traces, failures, and feedback into testable, regression-aware improvements that preserve prior capabilities.

- **Production failures outpace benchmark coverage.** The ways agents fail with real users are systematically underrepresented in pre-deployment evals, making a post-deployment learning loop necessary rather than optional.
- **Four core requirements structure the framework.** The approach is built around: converting failures into replayable learning environments; and preserving prior capabilities during updates — the remaining requirements fall outside the captured transcript excerpt.
- **Regression-awareness as a hard constraint.** Improvements derived from new failures must be validated against prior behavior before deployment — the framework makes this testable and explicit rather than assumed.
- **Production traces as learning material.** The system ingests real-world traces and feedback directly, bypassing the need for separately curated datasets to drive improvement.

**[MCP Apps: Primitives, discovery, and the Future of Software - Pietro Zullo, Manufact, Inc](https://www.youtube.com/watch?v=sAOBXCDiDOs)**
MCP Apps are not MCP servers with a UI grafted on. Pietro Zullo of Manufact frames them as a complete interaction layer — bidirectional, stateful, rendered by the host — where the model and UI share live context.

- **A distinct architectural category.** The defining properties — bidirectionality, statefulness, host rendering, and shared live context between model and UI — make MCP Apps qualitatively different from standard MCP servers with a frontend.
- **Shared live context changes interaction patterns.** Because the model and UI operate on the same live state, interaction patterns available to MCP Apps cannot be replicated by a conventional server-plus-client split.
- **Full lifecycle addressed.** The talk covers what MCP Apps are, how their architecture works, and how to build and distribute them — positioning them as a near-term primitive for software distribution.

**[Your AI Product Will Fail Unless You Can Explain It - Veronica Hylak, Hey AI](https://www.youtube.com/watch?v=d_Ftrl3vfV0)**
Shipping velocity no longer compensates for unclear value communication. Veronica Hylak argues that if average users can't instantly understand what an AI product does the product fails — and that this communication gap has become a primary failure mode for AI teams.

- **The discovery advantage is gone.** A decade ago the market would find a technically interesting product; today the first filter is whether a non-technical user can immediately grasp what it does.
- **The communication gap is now fatal.** Hylak frames the inability to quickly answer "why does this matter to an average user?" not as a marketing shortcoming but as a product-survival issue.
- **Practitioner-sourced perspective.** The talk draws on 7M views of content making complex AI systems legible to everyday people, and direct work with YC startups, AI tools, and safety organizations across SF.
- **One diagnostic test.** The central question — "why does this matter to an average user?" — is positioned as the litmus test every team should be able to answer before shipping.
