# Evolution Ideas

A scratchpad for ideas to evolve this project. Drop anything here — rough is fine.
Promote the good ones into issues or a `feat:` branch when you're ready.

## How to use

- Add a new idea under **Backlog** using the template below.
- When you start working on one, move it to **In progress**.
- When it ships, move it to **Done** with the commit/PR reference.

### Template

```
### <short title>
- **What:** one-line description of the idea
- **Why:** the problem it solves or value it adds
- **Notes:** sketches, links, open questions (optional)
- **Added:** YYYY-MM-DD
```

## Backlog

### Cross-reference digests against my studies
- **What:** A mechanism that periodically scans the digests I've received and
  checks them against the notes/ideas in the `studies/` folder — surfacing where
  a new article reinforces, updates, or *challenges* something I've written.
- **Why:** Keeps my studies alive instead of static. New industry reading should
  actively pressure-test my existing thinking (PM, Data Platform, Engineering),
  flagging outdated assumptions and ideas worth revisiting.
- **Notes:**
  - Inputs: accumulated `digests/` + the markdown in `studies/`.
  - Output: a periodic report — e.g. "Study X's claim about Y may be challenged
    by digest article Z" — with links back to both sources.
  - Open questions: how often to run (weekly?); scope to recent digests or all;
    where the report lands (new file, section in a digest, separate doc);
    how to match topics (LLM semantic match vs. tags/keywords).
- **Added:** 2026-06-18
